# 电流数据攻击分类实验设计表

## 实验目标

本轮实验主要验证以下问题：

1. 最优信息尺度是否在 `200` 点附近。
2. 准确率提升主要来自窗口长度，还是来自更小步长带来的更多训练样本。
3. 当前最优结果是否稳定，是否可能受到窗口高重叠带来的偏高影响。

## 固定条件

除窗口大小和步长外，其余训练条件尽量保持一致：

| 项目 | 固定设置 |
|---|---|
| 采样频率 | `1240 Hz` |
| 分类任务 | `normal / tcp / udp / icmp` 四分类 |
| 数据划分 | 先按原始时间序列划分训练/验证/测试，再分别滑窗 |
| 归一化 | 只用训练集统计量做 `Z-score` |
| 模型结构 | 固定同一版 `CNN-LSTM` |
| 训练轮数 | 固定 |
| 学习率 | 固定 |
| Batch Size | 固定 |

## 评价指标

每组实验建议统一记录以下指标：

| 指标 | 说明 |
|---|---|
| 训练窗口数 | 训练集滑窗后样本数 |
| 验证窗口数 | 验证集滑窗后样本数 |
| 测试窗口数 | 测试集滑窗后样本数 |
| 最佳验证准确率 | 训练过程中最佳验证结果 |
| 最终测试准确率 | 最终在测试集上的准确率 |
| Macro-F1 | 类别不平衡时更稳健 |
| 每类 Precision / Recall / F1 | 分析类别偏科问题 |
| 混淆矩阵 | 观察哪几类最容易混淆 |
| 训练时间 | 比较不同参数的成本 |
| 3 次重复实验均值和标准差 | 判断结果稳定性 |

## 判定规则

实验完成后，建议按以下逻辑判断结果：

| 观察现象 | 建议结论 |
|---|---|
| `200/20` 与 `200/32` 也明显表现强 | 最优窗口确实可能在 `200` 附近 |
| `256/16` 明显优于 `256/64` 或 `256/32` | 之前较差主要是步长过大导致样本不足 |
| `200/10` 明显高很多，但 `200/20` 和 `200/32` 下降明显 | 需要重点检查是否存在高重叠带来的偏高现象 |
| `160/32`、`200/32`、`220/32` 结果接近 | 最优区间可能不是单点，而是一段窗口范围 |
| 某一类召回率长期偏低 | 需要单独检查该类数据质量、采集条件或类别特征不足问题 |

## 最终汇总结论建议

最终不要只写“哪组准确率最高”，建议按照下面结构汇总结论：

| 结论项 | 示例写法 |
|---|---|
| 最优窗口区间 | `180-220` 点 |
| 最优步长区间 | `10-32` 点 |
| 是否依赖高重叠 | 是 / 否 |
| 是否稳定复现 | 是 / 否，给出 3 次重复均值和标准差 |
| 易混淆类别 | 例如 `udp` 与 `icmp` |
| 推荐默认参数 | 例如 `200 / 20` 或 `200 / 10` |
